# Triangle Product Rules

Before building product features, read:

- `VISION.md`
- `PRODUCT_OPERATING_RULES.md`
- `WORKFLOW_SIGNAL_TO_PLACEMENT.md`
- `RESEARCH_WORKBENCH.md`
- `ROADMAP_EXECUTION.md`
- `DECISIONS.md`
- `CURRENT_STATE.md`

Triangle Services is not building a generic CRM.
The product is a project-to-placement operating system.

The core workflow is:

`signal -> qualified project -> contractor chain -> buyer contacts -> crew package -> outreach -> opportunity -> placement`

Do not treat a discovered project, company record, news article, or AI summary as a win.
The win is a qualified project package opportunity with a plausible buyer, a specific crew package, and a next commercial action.

The project owner is usually not the labor buyer.

# Pitfalls we have already hit (don't repeat)

These are real bugs that have wasted time. Read before touching the matching area.

## Supabase / database

- **Org membership lookup must use the service client.** The cookie/anon client hits RLS and silently returns null, breaking every authed API route. Pattern is in `src/lib/supabase/server.ts → requireApiAccess`. Always: cookie client to verify the JWT, service client for the membership read.
- **Migrations are idempotent — but `CREATE TABLE IF NOT EXISTS` does NOT add new columns to a pre-existing table.** When adding fields to an existing table, also add `ALTER TABLE … ADD COLUMN IF NOT EXISTS …`. Drop and recreate is never the answer.
- **After ANY column or enum change, run `NOTIFY pgrst, 'reload schema';`** or PostgREST keeps serving the old schema. Symptom: `Could not find the 'X' column of 'Y' in the schema cache` even though the column is in `information_schema.columns`.
- **Enums are lowercase, AI output isn't.** The `chain_role` enum is `owner|developer|gc|epc|mep|electrical|intermediary|other`. AI happily returns `"MEP"`, `"OPERATOR"`, `"MEP / Electrical"`. Always normalize via `normalizeChainRole()` in `src/lib/data/research.ts` before INSERT.
- **`research_suggestions.source_url` is NOT NULL.** Notes generated by the AI often lack a specific URL. Fall back to `project.source_url`, then a placeholder. See `src/app/api/research/run/route.ts` notes branch.

## Next.js 16

- **Middleware lives in `src/proxy.ts`, not `middleware.ts`.** It refreshes the Supabase JWT and redirects unauthenticated requests. Any new path that needs auth-skipping (API routes, webhooks) must be added to the `isAuthPath` check or it will 302 to `/login`.

## OpenAI Responses API (tool calling)

- **Turn 0: pass `input` as a STRING.** Turn N+: pass an array of `{type:"function_call_output", call_id, output}` items + `previous_response_id`. Do NOT echo `function_call` items back — that's the duplicate-id 400 error.
- **Structured outputs drift.** GPT will return objects where strings are expected, floats where ints are expected, bare URL strings where `{source_url, evidence_text}` objects are expected. Build the Zod schema with `z.preprocess()` coercion and `safeParse` each array item so one bad row doesn't kill the whole response. Pattern is in `src/app/api/research/run/route.ts`.

## MCP / partner access

- **MCP server config goes in `.mcp.json` at project root**, NOT in `settings.json`. Claude Code's settings schema rejects top-level `mcpServers`.
- **Static API key auth must run BEFORE Supabase auth** in `requireApiAccess`. Pattern: check `MCP_API_KEY` first, then fall through to JWT/cookie auth. Required env vars: `MCP_API_KEY`, `MCP_ORGANIZATION_ID`, `MCP_USER_ID`.

## Research workbench

- **Suggestions are the source of truth, not final tables.** AI never writes to `contractor_chain_nodes` or `buyer_contacts` directly. It writes to `research_suggestions` (status `pending`). A human accept moves it to the final table via `acceptResearchSuggestion()`. Notes and package_opportunities stay in `research_suggestions` permanently.
